Графики появятся после первой зарядки и накопления ≥3 замеров. DiPlus snapshot'ы пишутся независимо от autoservice — этот блок будет работать у всех.
Секция А (период) — у всех, считается из нашей БД зарядок. Секция Б (Lifetime) — только при autoservice. Секция В (Динамика) — у всех (DiPlus snapshot'ы), но требует ≥3 замеров.
Триггер snapshot'а: при детекте пробуждения машины приложение запрашивает у DiPlus cell V min/max, темп. батареи, 12V, SoC и пишет одну запись в battery_snapshot. Триггерится при: (а) запуск сервиса после сна, (б) детекте окончания зарядки (SoC вырос), (в) первом запуске за день.

Резюме изменений

1. Унификация диалогов: Battery / Parking / AI Insights — единое поведение и позиционирование (CenterStart, тап-внутрь = закрытие).
2. Battery Health expand упрощён: только Блок «Сейчас» (DiPlus, у всех) + Блок «Lifetime» (autoservice или плейсхолдер). Без графиков, без расчётного SoH.
3. /Зарядки правая колонка: 3 секции — Период (у всех), Lifetime (autoservice), Динамика (snapshot'ы при пробуждении).
4. ChargeEdit: SOC основной ввод, кВт·ч computed read-only при заполненных SOC. Fallback для DC без SOC.
5. Backend bugfix: починить write-path для DiPlus snapshot'ов в battery_snapshot table — сейчас Δ ячеек/темп. live идут, но не персистятся, поэтому графики «Динамика» пустые.
